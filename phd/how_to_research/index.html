How I Do Research (Working Playbook)

This document exists to reduce friction, not to impose discipline. Research sessions should start cleanly, end safely, and leave evidence behind.

This is a living document.

Core Principles

  • Research happens in sessions, not moods
  • Confusion is allowed; loss of state is not
  • Everything important must survive power loss, human error, and future-me
  • Streaming adds cognitive load — compensate explicitly

Before a Research Session (5–10 minutes)

Declare intent

Write one sentence answering:

What am I trying to figure out right now?

This can go in: - a research log - a scratchpad - the stream description - a comment at the top of a notebook

If I can’t state intent, I’m not ready to start.

Establish scope

Define: - what is in scope for this session - what is explicitly out of scope

This prevents productive-looking wandering.

Prep the environment

  • Open only the tools relevant to the session
  • Close unrelated tabs
  • Start recording only if this session benefits from being streamed

If streaming: - say the intent out loud - accept that pauses and mistakes are part of the artifact

During a Research Session

Think out loud (especially if streaming)

Narration forces clarity. Confusion spoken aloud is often halfway solved.

Capture decisions, not everything

Record: - assumptions - choices - changes in direction - things that didn’t work and why

Do not attempt to transcribe reality.

Leave breadcrumbs

If you stop mid-thought: - leave a TODO - write a one-line “next step” - add a comment explaining current state

Future-me should never have to reverse-engineer intent.

Writing (Lightweight Guidance)

Writing usually comes naturally, but structure still helps.

When to write

  • as soon as an idea stabilizes
  • before the research feels “done”

Drafts are thinking tools, not deliverables.

How to write productively

  • start ugly
  • write explanations as if to an intelligent outsider
  • prefer clarity over cleverness
  • use headings early; polish later

If stuck: - write the section I don’t want to write - explain the problem instead of the solution

Ending a Research Session (Non-Negotiable)

This is the most important part.

Write a closing note

Answer: - what did I learn? - what changed? - what is the next obvious step?

One paragraph is enough.

Commit meaningful changes

If files changed in a way that affects meaning:

git add . git commit -m "Describe what changed and why"

Commits are semantic checkpoints, not perfection markers.

Backup

Run the backup process. Do not defer this.

Backups should be: - encrypted - automated where possible - boring

If I’m too tired to back up, I’m too tired to stop safely.

Clean exit

  • Stop recording
  • Close tools
  • Leave workspace in a known-good state

Research sessions should end intentionally, not by exhaustion.

Streaming-Specific Notes

  • Streams are secondary artifacts
  • Clarity beats performance
  • Pauses and dead ends are allowed
  • Narrate uncertainty honestly

If a session shouldn’t be streamed, don’t force it.

Maintenance

Revisit this document: - when friction appears - after major workflow changes - when something breaks

The process exists to serve the work, not the other way around.
